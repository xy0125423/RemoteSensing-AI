# PROJECT_OVERVIEW — 项目总览

> 本文档是 RemoteSensing-AI 项目的总纲。  
> 阅读对象：未来的你、复试导师、任何接手本项目的人。

---

## 1. 项目名称

**RemoteSensing-AI — 遥感自动化分析系统**

---

## 2. 一句话描述

输入一个地名 → 自动获取 Sentinel-2 影像 → NDVI 时序分析 → 傅里叶频谱 → 自动生成 PDF 报告。

全部在 CPU 上运行，不依赖 GPU，不涉及深度学习。

---

## 3. 项目背景

### 3.1 我是谁

- 遥感科学与技术专业本科生
- 数学基础较好，正在准备考研（892 信号与线性系统）
- 已有 Python 基础，能使用 VS Code + Git
- 无 NVIDIA GPU，开发环境为 Lenovo 小新16 笔记本

### 3.2 为什么做这个项目

| 维度 | 说明 |
|------|------|
| **考研复试** | 西安电子科技大学遥感方向 — 需要一个能展示工程能力的作品 |
| **就业作品集** | 遥感开发岗需要证明"能独立完成端到端项目" |
| **工程能力** | 从"会用 API"到"能设计系统"的跨越 |
| **知识串联** | 把课堂上的遥感原理、信号与系统、编程真正串起来 |

### 3.3 为什么不做深度学习

1. **硬件限制**：无 GPU，笔记本跑 CNN 不现实
2. **学习阶段**：本科阶段应先吃透遥感基础（指数、时序、频谱），再接触 DL
3. **性价比**：Sentinel-2 + NDVI + FFT 已经能产出高质量分析报告，不需要 DL 也能出成果
4. **复试友好**：传统遥感方法能讲清楚"为什么"，DL 方法容易被追问细节

## 4. 最终目标

### 4.1 用户视角

```
用户输入：山东省菏泽市郓城县
         ↓
系统自动：
  ① GEE 获取 2025 年 Sentinel-2 影像
  ② 云量过滤（<10%）
  ③ 逐景计算 NDVI
  ④ 构建 NDVI 时间序列
  ⑤ 傅里叶变换分析植被变化周期
  ⑥ 生成：地图 + 折线图 + 频谱图 + 统计表

全流程一键运行，无需人工干预。
```

## 5. 技术路线

### 5.1 技术选型总览

| 层 | 技术 | 选型理由 |
|----|------|----------|
| **数据获取** | Google Earth Engine (Python API) | 免费、无需下载原始影像、云端计算 |
| **地理处理** | rasterio + shapely + geopandas | 轻量、CPU 友好、行业标准 |
| **数值计算** | numpy + scipy | scipy 提供 FFT，numpy 提供数组基础 |
| **可视化** | matplotlib + cartopy（可选） | 最通用、文档丰富、复试可展示 |
| **PDF 生成** | fpdf2 或 ReportLab | 纯 Python、轻量、可嵌入图片 |
| **项目管理** | Git + VS Code | 已有、熟练 |

### 5.2 为什么选 GEE 而不是本地下载 Sentinel-2

| 对比维度 | GEE | 本地下载 |
|----------|-----|----------|
| 存储占用 | 0（云端） | 单景 ~1GB，28 景 = 28GB |
| 计算位置 | Google 服务器 | 你的笔记本 CPU |
| 预处理 | 内置云掩膜、缩放 | 需要自己写 |
| 学习成本 | 中等（JS→Python） | 高（GDAL 生态） |
| 适合阶段 | ✅ 本科项目 | 生产环境/大量定制 |

### 5.3 为什么用 scipy.fft 而不是自己写 FFT

1. 你正在备考 892 信号与线性系统 — scipy.fft 的输出可以直接对应课本理论
2. 遥感中的 FFT 应用不需要自己实现算法，需要的是理解频谱含义
3. scipy 的 FFT 实现经过高度优化，CPU 上也能快速运行

## 7. 设计哲学

### 7.1 四大原则

1. **能跑 > 完美**  
   每个 Phase 结束，项目都是可运行的。绝不出现"写到一半卡住三个月"。

2. **模块化 > 一体化**  
   每个模块独立可测。`ndvi_calc.py` 不依赖 `report_gen.py`。

3. **文档 > 记忆**  
   所有决策写入 DECISIONS.md，所有踩坑写入 NOTES.md。六个月后你还能看懂。

4. **简单 > 炫技**  
   能用 for 循环就不用 generator，能用 list 就不用 numpy 2D 高级切片。可读性优先。

### 7.2 学习优先、工程其次

这个项目的首要目标是**你学会遥感AI开发的完整流程**，其次才是产出一个能用的工具。

因此：
- 每个模块你都要**手写核心逻辑**（如 NDVI = (NIR-Red)/(NIR+Red)），而不是调一个黑盒函数
- 每个阶段你都要**理解为什么**（如为什么用 FFT 而不是直接看折线图找周期）
- 速度优化、代码优雅是 Phase 5 之后的事
